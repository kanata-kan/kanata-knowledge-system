## File Hierarchy

```
.env                 ← default (all environments)
.env.local           ← local overrides (gitignored)
.env.development     ← dev only
.env.production      ← production only
.env.test            ← test only
```

Priority: `.env.local` > `.env.[environment]` > `.env`

## Public vs Private

```bash
# Private (server only) — never exposed to browser
DATABASE_URL=mongodb://...
JWT_SECRET=supersecret

# Public (exposed to browser) — prefix with NEXT_PUBLIC_
NEXT_PUBLIC_API_URL=https://api.example.com
NEXT_PUBLIC_APP_NAME=MyApp
```

## Type-Safe Access

```typescript
// lib/env.ts
import { z } from 'zod'

const envSchema = z.object({
  DATABASE_URL: z.string().url(),
  JWT_SECRET: z.string().min(32),
  NEXT_PUBLIC_API_URL: z.string().url(),
})

export const env = envSchema.parse({
  DATABASE_URL: process.env.DATABASE_URL,
  JWT_SECRET: process.env.JWT_SECRET,
  NEXT_PUBLIC_API_URL: process.env.NEXT_PUBLIC_API_URL,
})
```

## Usage

```typescript
// Server Component or API route
import { env } from '@/lib/env'

const db = connect(env.DATABASE_URL) // ✓ type-safe, validated
```

## Key Rules

- Never commit `.env.local` (add to `.gitignore`)
- `NEXT_PUBLIC_` vars are inlined at build time (not runtime)
- Server-only vars are only available in Server Components, Route Handlers, middleware
- Validate all env vars at startup (fail fast)
- Use different values per environment via hosting platform (Vercel, etc.)
