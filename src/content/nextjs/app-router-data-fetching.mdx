## Server Component Data Fetching

In the App Router, every component is a Server Component by default. You can fetch data directly inside components using async/await.

```typescript
// app/users/page.tsx
export default async function UsersPage() {
  const res = await fetch('https://api.example.com/users')
  const users = await res.json()

  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}
```

## Request Deduplication

React automatically deduplicates `fetch` requests with the same URL and options during a single render pass.

```typescript
// Both of these only make ONE actual network request
async function UserName({ id }: { id: string }) {
  const res = await fetch(`/api/users/${id}`)
  const user = await res.json()
  return <span>{user.name}</span>
}

async function UserEmail({ id }: { id: string }) {
  const res = await fetch(`/api/users/${id}`)
  const user = await res.json()
  return <span>{user.email}</span>
}
```

## Caching Strategies

```typescript
// Cached indefinitely (default in production)
fetch('https://api.example.com/data')

// Revalidate every 60 seconds
fetch('https://api.example.com/data', {
  next: { revalidate: 60 }
})

// Never cache — always fresh
fetch('https://api.example.com/data', {
  cache: 'no-store'
})
```

## Parallel Data Fetching

Always fetch in parallel when data is independent:

```typescript
export default async function Dashboard() {
  // GOOD: parallel
  const [users, posts, analytics] = await Promise.all([
    getUsers(),
    getPosts(),
    getAnalytics(),
  ])

  return <DashboardView users={users} posts={posts} analytics={analytics} />
}
```
