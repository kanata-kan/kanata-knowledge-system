## Installation

```bash
npm install clsx tailwind-merge
```

## The Utility

```typescript
// lib/utils.ts
import { type ClassValue, clsx } from 'clsx'
import { twMerge } from 'tailwind-merge'

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

## Usage

```typescript
import { cn } from '@/lib/utils'

// Conditional classes
<div className={cn(
  "px-4 py-2 rounded",
  isActive && "bg-accent text-white",
  isDisabled && "opacity-50 cursor-not-allowed"
)} />

// Merge without conflicts
cn("px-4 py-2", "px-6")
// → "px-6 py-2" (tailwind-merge resolves conflict)

// Component variants
function Button({ variant, className, ...props }) {
  return (
    <button
      className={cn(
        "px-4 py-2 rounded font-medium",
        variant === "primary" && "bg-accent text-background",
        variant === "ghost" && "hover:bg-border/50",
        className // allow consumer overrides
      )}
      {...props}
    />
  )
}
```

## Why Both Libraries?

- **clsx** — Conditional class joining (handles arrays, objects, falsy values)
- **tailwind-merge** — Resolves Tailwind class conflicts (last wins)
- Together = reliable className composition without duplicates
