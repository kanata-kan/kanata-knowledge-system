## Cache Layers

```
Browser Cache → CDN Edge → Next.js Cache → Origin Server → Database
```

Each layer reduces load on the next.

## Browser Cache Headers

```typescript
// Static assets (immutable hash in filename)
Cache-Control: public, max-age=31536000, immutable

// API responses (revalidate)
Cache-Control: public, max-age=0, s-maxage=60, stale-while-revalidate=300

// Private user data
Cache-Control: private, no-cache
```

## Next.js Data Cache

```typescript
// Cache indefinitely (default)
fetch('https://api.example.com/data')

// Time-based revalidation
fetch('https://api.example.com/data', {
  next: { revalidate: 3600 } // 1 hour
})

// On-demand revalidation
import { revalidateTag } from 'next/cache'

fetch('https://api.example.com/posts', {
  next: { tags: ['posts'] }
})

// In Server Action after mutation:
revalidateTag('posts')
```

## Stale-While-Revalidate Pattern

Serve stale content immediately, fetch fresh in background:

```
Request → Is cached? → Yes → Serve cached
                        ↓
                   Is stale? → Yes → Revalidate in background
```

## Cache Invalidation Strategies

1. **Time-based** — TTL expires, refetch
2. **Event-based** — Mutation triggers revalidation
3. **Version-based** — Content hash changes
4. **Manual** — Admin triggers purge

## Key Rules

- Cache static assets aggressively (immutable)
- Cache API responses with short TTL + SWR
- Never cache user-specific data on shared CDN
- Use `revalidateTag` for targeted invalidation
- Monitor cache hit rates
