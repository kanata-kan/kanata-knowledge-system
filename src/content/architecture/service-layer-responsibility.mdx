## Core Principle — المبدأ الأساسي

Service Layer مسؤولة فقط على:

- **business logic** — القواعد اللي تحكم البيانات
- **data transformation** — تحويل raw data لشكل يفهمه التطبيق
- **normalization** — توحيد البيانات من مصادر مختلفة
- **orchestration** — تنسيق بين services متعددة
- **formatting** — تنسيق القيم قبل العرض
- **reusable operations** — عمليات مشتركة بين أكثر من feature

> **"العقل التشغيلي للتطبيق — بدون UI."**

## Wrong vs Correct — مقارنة عملية

### ❌ Wrong — logic داخل Component

```tsx
export function UserCard({ user }) {
  // ❌ transformation داخل UI
  const fullName = `${user.first_name} ${user.last_name}`;
  const isVerified = user.status === "verified" && user.email_confirmed;
  const joinedDate = new Date(user.created_at).toLocaleDateString("ar-MA");

  return (
    <div>
      <p>{fullName}</p>
      <p>{joinedDate}</p>
      {isVerified && <Badge>Verified</Badge>}
    </div>
  );
}
```

> **المشكل:** Component ولات مسؤولة على UI + transformation + business rules في نفس الوقت.

---

### ✅ Correct — كل طبقة في مكانها

**`services/userService.ts`** — transformation هنا فقط

```ts
export function mapUser(raw) {
  return {
    fullName: `${raw.first_name} ${raw.last_name}`,
    isVerified: raw.status === "verified" && raw.email_confirmed,
    joinedDate: new Date(raw.created_at).toLocaleDateString("ar-MA"),
  };
}
```

**`hooks/useUser.ts`** — fetch + mapping

```ts
import { mapUser } from "@/services/userService";

export async function useUser(id: string) {
  const res = await fetch(`/api/users/${id}`);
  const raw = await res.json();
  return mapUser(raw);
}
```

**`components/UserCard.tsx`** — UI فقط، بلا logic

```tsx
export function UserCard({ user }) {
  return (
    <div>
      <p>{user.fullName}</p>
      <p>{user.joinedDate}</p>
      {user.isVerified && <Badge>Verified</Badge>}
    </div>
  );
}
```

## Testing Advantage — سهولة الـ Testing

Service Layer pure functions = تستعملها بدون React أو browser:

```ts
// ✅ unit test بسيط — بدون rendering، بدون browser
expect(mapUser(rawUser).fullName).toBe("Abdelilah Wajid");
expect(mapUser(rawUser).isVerified).toBe(true);
```

## Common Mistakes — أخطاء شائعة

### 1. Business Logic داخل Component

```tsx
// ❌ logic كبيرة مكانها service
const total = items.reduce((sum, item) => sum + item.price * item.qty, 0);
const discounted = total > 500 ? total * 0.9 : total;
```

---

### 2. API Mapping داخل Hook

```ts
// ❌ transformation مكانها service
const formatted = {
  name: res.data.user_name,
  avatar: res.data.profile_picture_url,
};
```

---

### 3. Analytics داخل Service Layer

```ts
// ❌ analytics concern مستقل — ما تخلطوش
export function mapOrder(raw) {
  trackEvent("order_mapped"); // ❌
  return { ... };
}
```
