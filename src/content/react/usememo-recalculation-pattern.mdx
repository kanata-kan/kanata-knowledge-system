## Problem — المشكل

في React، أي component كتعاود render:

- functions كتعاود تتنفذ
- calculations كتعاود تتحسب
- objects كتتخلق من جديد

حتى إلا البيانات نفسها ما تبدلاتش — هاد الشي ممكن يخلق:

## Core Principle — المبدأ الأساسي

`useMemo` كتخزن نتيجة calculation وتعاود تستعملها حتى تتبدل dependencies.

> **"إلى المعطيات ما تبدلاتش، استعمل النتيجة القديمة."**

## Basic Example — مثال بسيط

### ❌ Without useMemo

```tsx
function Dashboard({ users }) {
  const activeUsers = users.filter((user) => user.active);

  return <UsersList users={activeUsers} />;
}
```

> **المشكل:** كل render → `filter()` كتعاود تخدم حتى إلا `users` ما تبدلاتش.

---

### ✅ With useMemo

```tsx
const activeUsers = useMemo(() => {
  return users.filter((user) => user.active);
}, [users]);
```

> **النتيجة:** calculation كتخدم فقط عند تغير `users` — باقي renders كيستعملو cached value.

---

## Mental Model — النموذج العقلي

`useMemo` هي **cache صغيرة داخل component** — لكن:

- مؤقتة
- مرتبطة بالـ render lifecycle
- ليست global cache

## Stable Reference Problem — مشكلة الـ Reference

### ❌ Wrong

```tsx
const config = {
  dark: true,
};
```

> كل render → object جديد حتى إلا القيمة نفسها — يكسر `React.memo()` على children.

---

### ✅ Correct

```tsx
const config = useMemo(() => {
  return {
    dark: true,
  };
}, []);
```

> Reference ثابتة = children ما كيعاودوش render بلا سبب.

---

## Common Mistakes — أخطاء شائعة

### 1. useMemo Everywhere

```tsx
// ❌ useless — string ثابتة لا تحتاج memo
const name = useMemo(() => "abdelilah", []);
```

---

### 2. Confusing useMemo With Performance Magic

`useMemo` **ليست** performance booster سحرية — هي فقط **تمنع إعادة الحساب غير الضرورية**.

---

### 3. Missing Dependencies

```tsx
// ❌ خطأ — data مش فـ dependencies
useMemo(() => {
  return calculate(data);
}, []);
```

> إذا `data` تتبدل، النتيجة غادي تولي **stale**.

---

## Decision Rule — قاعدة اتخاذ القرار

اسأل نفسك:

> "هل هاد الـ calculation فعلاً مكلفة أو كتسبب re-renders غير ضرورية؟"

- **نعم** → `useMemo` مناسبة
- **لا** → لا تستعملها

## Real World Usage — الاستخدام الحقيقي

`useMemo` شائعة بزاف في:

- Dashboards و analytics systems
- Data-heavy tables و filtering systems
- Charts و expensive data transformations

### القاعدة السريعة

> **"كتحتاج قيمة؟ → `useMemo` — كتحتاج تنفذ شي؟ → `useEffect`"**

---

### مثال يوضح الفرق

#### ❌ خطأ شائع — استعمال useEffect لحساب قيمة

```tsx
const [activeUsers, setActiveUsers] = useState([]);

useEffect(() => {
  // ❌ هاد مش side effect — هي مجرد calculation
  setActiveUsers(users.filter((u) => u.active));
}, [users]);
```

المشكل:

- render إضافي بسبب `setState`
- كود معقد بلا سبب
- `activeUsers` كتولي stale لحظة واحدة قبل update

---

#### ✅ الصح — استعمل useMemo مباشرة

```tsx
const activeUsers = useMemo(() => {
  return users.filter((u) => u.active);
}, [users]);
```

نتيجة synchronous مباشرة — بلا re-render إضافي.

---

#### ✅ useEffect في مكانه الصحيح

```tsx
useEffect(() => {
  // ✅ هاد side effect حقيقي — fetch من API
  fetch("/api/users")
    .then((res) => res.json())
    .then((data) => setUsers(data));
}, []);
```
