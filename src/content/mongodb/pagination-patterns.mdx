## Skip/Limit (Offset-Based)

Simple but degrades at scale:

```typescript
async function getPage(page: number, limit: number = 20) {
  const skip = (page - 1) * limit

  const [items, total] = await Promise.all([
    db.collection('posts')
      .find({})
      .sort({ createdAt: -1 })
      .skip(skip)
      .limit(limit)
      .toArray(),
    db.collection('posts').countDocuments({}),
  ])

  return {
    items,
    pagination: {
      page,
      limit,
      total,
      totalPages: Math.ceil(total / limit),
      hasNext: page * limit < total,
    },
  }
}
```

## Cursor-Based (Recommended)

Uses last document's sort value as cursor:

```typescript
type CursorPage<T> = {
  items: T[]
  nextCursor: string | null
  hasMore: boolean
}

async function getCursorPage(cursor?: string, limit: number = 20): Promise<CursorPage<Post>> {
  const query: any = {}

  if (cursor) {
    query.createdAt = { $lt: new Date(cursor) }
  }

  const items = await db.collection('posts')
    .find(query)
    .sort({ createdAt: -1 })
    .limit(limit + 1) // fetch one extra to detect hasMore
    .toArray()

  const hasMore = items.length > limit
  if (hasMore) items.pop() // remove extra item

  const nextCursor = hasMore
    ? items[items.length - 1].createdAt.toISOString()
    : null

  return { items, nextCursor, hasMore }
}
```

## When to Use Which

| Pattern | Pros | Cons |
|---------|------|------|
| Skip/Limit | Simple, page numbers | Slow at high offset, inconsistent on insert |
| Cursor | Fast at any depth, consistent | No random page access |

## Performance Rule

Skip/Limit scans and discards `skip` documents. At page 1000 with limit 20, MongoDB scans 20,000 docs.

Cursor-based uses an index to start at exactly the right point — O(1) regardless of "page depth".
