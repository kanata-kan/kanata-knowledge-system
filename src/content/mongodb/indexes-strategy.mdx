## Index Types

### Single Field Index

```typescript
db.collection.createIndex({ email: 1 })  // ascending
db.collection.createIndex({ score: -1 }) // descending
```

### Compound Index

```typescript
// Supports queries on: status, status+createdAt, status+createdAt+priority
db.collection.createIndex({ status: 1, createdAt: -1, priority: 1 })
```

### Text Index

```typescript
db.collection.createIndex({ title: 'text', content: 'text' })

// Query
db.collection.find({ $text: { $search: 'mongodb aggregation' } })
```

## ESR Rule (Equality, Sort, Range)

Order compound index fields by:

1. **Equality** — Fields tested with exact match (`status: 'active'`)
2. **Sort** — Fields used in sort (`createdAt: -1`)
3. **Range** — Fields with range queries (`age: { $gte: 18 }`)

```typescript
// Query: find active users, sort by date, age > 18
db.users.find({ status: 'active', age: { $gte: 18 } }).sort({ createdAt: -1 })

// Optimal index follows ESR:
db.users.createIndex({ status: 1, createdAt: -1, age: 1 })
```

## Common Mistakes

- Too many indexes slow down writes
- Unused indexes waste memory — audit regularly
- Index prefix matters: `{ a: 1, b: 1 }` supports queries on `a` but NOT queries on only `b`
- Don't index fields with low cardinality (e.g., boolean) alone
