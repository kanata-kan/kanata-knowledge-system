## Recommended Base Config

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "esnext",
    "moduleResolution": "bundler",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx"
  }
}
```

## Key Options Explained

| Option | Purpose |
|--------|---------|
| `strict` | Enables all strict checks (always use) |
| `moduleResolution: "bundler"` | Modern resolution for Vite/Next.js |
| `skipLibCheck` | Skip checking .d.ts files (faster builds) |
| `isolatedModules` | Required for tools like esbuild/SWC |
| `noEmit` | Don't emit JS (let bundler handle it) |

## Path Aliases

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"],
      "@/components/*": ["./src/components/*"]
    }
  }
}
```

## Strict Mode Sub-Options

When `strict: true` is enabled, it includes:

- `strictNullChecks` — null/undefined are distinct types
- `strictFunctionTypes` — stricter function type checking
- `strictBindCallApply` — check bind/call/apply
- `strictPropertyInitialization` — class properties must be initialized
- `noImplicitAny` — error on implicit `any`
- `noImplicitThis` — error on implicit `this`
- `alwaysStrict` — emit `"use strict"`
